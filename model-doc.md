## 三、核心公式

### SetRules（多输入单公式）

#### S1: 房租 B5 = f(B14面积, B15等级)
```
a = B14(面积), g = B15(等级)
B5 = max(0, round(a × g × max(2, 20 − a × 0.05)))
```
说明：非线性折扣，面积越大每平米单价越便宜。200m²时单价打5折（20→10），小面积几乎不打折。

#### S2: 需求 B2 = f(B1价格, B15等级, B13营销, B17缺货惩罚)
```
p = B1(价格), g = B15(等级), m = B13(营销), s = B17(缺货率)

priceEffect   = max(300, 5000 − p × 200)    // 价格越低需求越大，有底线300
locationEffect = g × 200                      // 每1等级+200需求
marketingEffect = √max(0, m) × 15            // 营销效应：边际递减（开根号）

base    = priceEffect + locationEffect + marketingEffect
penalty = round(base × s × 0.5)             // 每缺货1%赶走0.5%客户

B2 = round(base − penalty)                   // 最终需求 >= 0
```

#### S3: 加工成本 B4 = f(B3产能)
```
B4 = max(0.1, 2 − B3 × 0.0002)
```
说明：产能越高加工成本越低。下限0.1元/个（原0.5元），斜率加倍。B3=5000时B4=1.0（原1.5）。

### 纠缠边（循环依赖 + 冲突仲裁 + 动态备货系数）

所有纠缠边运行时，同一节点收到多个提案时，**权重最高的获胜**（Ghost仲裁）。
所有向 B3 写提案的纠缠边均使用**动态备货系数**替代固定 1.2 倍：

```
上期缺货率 = B17
上期报废率 = B18
备货信心系数 = max(0.6, min(1.4, 1.0 + B17×0.5 − B18×0.5))

如果上期有缺货（B17>0）：信心↑，多备货（接近 1.4）
如果上期有报废（B18>0）：信心↓，少备货（接近 0.6）
理想情况（B17=B18=0）：信心=1.0，刚好补足
```

#### E1: B16→B3（权重10 — 产能计划主要驱动力）
```
area  = B14
labor = B9
lastDemand = B16

如果 area<=0 或 labor<=0: B3=0

baseFromArea  = floor(area × 25)       // 每m²可放25个面包
baseFromLabor = floor(labor / 5.0)      // 每5元人工出1个产能（原7.2元）
resourceCap   = min(baseFromArea, baseFromLabor) // 短板效应

B3 = min(resourceCap, round(lastDemand × conf))  // 按上期需求×动态系数备货
```

#### E2: B4→B3（权重7 — 成本效率红利）
```
area  = B14
labor = B9
lastDemand = B16
fromDemand = round(lastDemand × conf)

如果 area<=0 或 labor<=0: B3=0

baseFromArea  = floor(area × 25)
baseFromLabor = floor(labor / 5.0)

costBoost = max(0, (2 − B4) × 200)      // 加工成本每低于2元1分→+2产能
effectiveCap = floor(min(baseFromArea, baseFromLabor) + costBoost)

B3 = min(effectiveCap, fromDemand)
```

#### E3: B9→B3（权重8 — 人工变化→资源上限）
```
area  = B14
labor = B9
lastDemand = B16

如果 area<=0 或 labor<=0: B3=0

resourceCap = min(floor(area × 25), floor(labor / 5.0))

B3 = min(resourceCap, round(lastDemand × conf))
```

#### E4: B14→B3（权重8 — 面积变化→资源上限）
```
area  = B14
labor = B9
lastDemand = B16

如果 area<=0 或 labor<=0: B3=0

resourceCap = min(floor(area × 25), floor(labor / 5.0))

B3 = min(resourceCap, round(lastDemand × conf))
```

### 衍生公式（MeshFlow公式引擎，无需手动推演）

| 节点 | 公式 | 说明 |
|------|------|------|
| **B12** | `= (B10 + B11 + B4) × B3` | 总生产成本 |
| **B6** | `= B1 × MIN(B2, B3)` | 月收入 = 售价 × 实际销量（产能瓶颈） |
| **B7** | `= B12 + B5 + B9 + B13` | 总成本 = 生产 + 房租 + 人工 + 营销 |
| **B8** | `= B6 − B7` | 月利润 |

## 四、月度推进（点击「📅 下个月」）

```
demand  = B2(当前需求)
cap     = B3(当前产能)
revenue = B6(当前月收入)
cost    = B7(当前月成本)
profit  = B8(当前月利润)

1. 记录本月数据到年度历史
2. 累加: 年利润 += profit, 年收入 += revenue, 年成本 += cost
3. 计算缺货率:
   shortage = (cap>=demand || demand<=0) ? 0 : round((demand-cap)/demand × 1000) / 1000
4. 计算报废率:
   waste = (cap>demand && cap>0) ? round((cap-demand)/cap × 1000) / 1000 : 0
5. 快照: B16 = demand, B17 = shortage, B18 = waste
6. 触发全引擎传播（SetRules + 纠缠收敛）
7. 月份+1
```

## 五、推演流程（单次传播顺序）

1. **输入变化** — B1(售价) / B14(面积) / B15(等级) / B9(人工) / B13(营销)
2. **S1: 房租** — B14×B15→B5
3. **缓存就绪** — B16(上期需求) / B17(缺货率)
4. **S2: 需求** — B1×B15×B13−B17→B2
5. **E1+E3+E4: 产能计划** — B16×资源→B3（B16权10, B9权8, B14权8 同时竞选）
6. **S3: 加工成本** — B3→B4
7. **E2: 成本红利** — B4→B3（权7，与E1冲突但同轮内B16权10胜出）
8. **衍生: 生产成本** — B12 = (B10+B11+B4)×B3
9. **衍生: 收入** — B6 = B1×MIN(B2,B3)
10. **衍生: 总成本** — B7 = B12+B5+B9+B13
11. **衍生: 利润** — B8 = B6−B7

## 六、仿真沙盘规则

- **一年12个月**，点击「📅下个月」推进
- 每步记录：月份、本月利润、本月收入、本月成本、需求、实际销量
- B17(缺货率)影响下月B2(需求)：上期没买到的客人在当期流失一半
- 年总结：总收入、总成本、总利润、利润率、月均利润、盈利月数
- 点击「🔄新一年」重置所有计数器

## 七、已知数学特性

- **线性成本关系**：房租有人均折扣，但面积→产能(25)和人工→产能(5.0)均为线性，资源上限取二者短板
- **冲突仲裁**：B3同时接收E1(权10)、E2(权7)、E3(权8)、E4(权8)提案，同轮内B16(权10)胜出；下轮如B16未变则E2(成本红利)可生效
- **成本底线**：原料3+其他1+加工0.1+人工5.0+房租折扣后→精品店约14.3元/个，工厂约12.0元/个
- **两条策略路线**：
  - 精品模式：¥24定价，46m²，精益人工 → 高毛利低量
  - 工厂模式：¥14定价，200m²，规模人工 → 薄利多销
