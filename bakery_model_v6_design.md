# 面包店经营沙盘 — 数学模型设计文档 v6

## 概述

玩家经营一家虚拟面包店。每月调整 8 个滑块（售价/人数/培训/营销/面积/地段/原料/工资），点击「推进下月」看盈亏。核心机制是 28+ 节点的依赖图，由 MeshFlow 引擎实时传播求值。三条路线：社区店、中央工厂、高奢精品。

## 二、核心计算公式

### B9 — 人工成本

```
B9 = round( 员工数 × 工资 × (1 + 地段 × 0.15 × max(0, 1 - 员工数 × 0.08)) )
```

- 地段加成：好地段招人贵（B15×0.15）
- 管理衰减：人多后边际效率下降（1-员工数×0.08），大于12人后系数<0.04

### B5 — 租金

```
商业区 (地段 ≥ 4):  rate = max(18, 50 - 面积 × 0.07)
工业区 (地段 ≤ 3):  rate = max(5,  40 - 面积 × 0.07)

B5 = round( 面积 × 地段 × rate )
```

- 商业区 60m² 地段4：60×4×45.8 = ¥10,992
- 商业区 200m² 地段4：200×4×36 = ¥28,800
- 工业区 150m² 地段2：150×2×29.5 = ¥8,850
- 工业区 250m² 地段2：250×2×22.5 = ¥11,250
- 面积越大每平米越便宜，但有底线

### B3 — 月产能

```
physCap = min( 面积 × 30,  员工 × 1500 ) × wf
wf = 0.2 + 1.6 × min(工资, 10000) / 10000   // 工资¥10k时 wf=1.8

FAT 系数 sk(FAT):
  FAT < 80:   1.0
  FAT ≥ 100:  0.1
  80–100:     sigmoid 平滑衰减 (t³型, t=(FAT-80)/20)

B3 = round( physCap × sk(FAT) )
```

产能受面积或人工的短板限制：
- 60m² + 3人 ¥4k工资：min(1800, 4500) × 0.84 = 1512 件
- 150m² + 3人 ¥3.5k工资：min(4500, 4500) × 0.76 = 3420 件
- 200m² + 5人 ¥4k工资：min(6000, 7500) × 0.84 = 5040 件

### B2 — 月需求（零售客流）

```
pbF (价格弹性):
  B1 < 20:  1 + (20 - B1) × 0.15
  B1 ≥ 20:  max(0.6, 1 - (B1 - 20) × 0.03)

入店客流 = 地段自然流 + 店面可见度 + 品牌口碑 + 品质复购

地段自然流  = (450 + 400 × min(B15, 6)) × pbF     ← B15>6后软天花板
店面可见度  = round(√面积 × 15) × pbF             ← 大店面天然广告牌
品牌口碑    = round(BRAND × 1.2) × pbF
品质复购    = round(B28 × 250) × pbF

ma (最大可接受价) = max(1, 15 + B15 × 2 + min(BRAND × 0.35, 60) + B21 × 3
                     + max(0, (B26-4000)/500) + B28 × 4)

转化率:
  B1 ≤ ma:  min(0.9, 0.5 + (ma - B1) / ma × 0.4)   ← 定价在承受范围内
  B1 > ma:  max(0.05, 0.5 × ma / B1)                 ← 定价过高, 惩罚

季节系数 SZ = [0.85, 1.10, 0.95, 1.00, 1.00, 0.90, 0.85, 0.85, 1.25, 1.10, 1.00, 1.30]
品牌护城河: BRAND≥200 → 淡季不低于0.95; BRAND≥100 → 不低于0.90

零售需求 = max(0, round( 入店客流 × 转化率 × 季节(护城河修正) ))

旅游客流 (B15 ≥ 5 触发):
  tourBase = (B15 - 4) × 600 × pbF
  mktEffect = round(√营销 × 6) × (0.5 + BRAND/400)
  tourMa = 10 + B15×2 + 营销/3000 + qpV + BRAND×0.3
  转化率: 0.40–0.80
  tourDemand = round( (tourBase + mktEffect) × tourConv × sz )

B2 = 零售需求 + 旅游客流
```

### B6 — 实际销量

```
B6 = min(B2, B3)
```

### B7 — 月营收

```
零售收入 = B6 × B1 + round(B6 × B1 × 0.20)     // 含20%溢价(包装/服务)

工厂 B2B 收入 (面积 ≥ 150 触发):
  B2B 合同量 = min(B24 × 500,  剩余产能)         // 每人月接500件企业订单
  冗余批发量 = min((B14-150) × 200,  剩余产能)   // 多余产能走批发行情
  wsSold = min(B2B合同量 + 冗余批发量,  剩余产能)
  wsRev = round(wsSold × B1 × 0.45)               // 批发价=零售价45%

B7 = 零售收入 + wsRev
```

### B8 — 月利润

```
B4 (加工成本) = max(0.1,  (2 - B3 × 0.0002) × (1 - EMP × 0.002)
                        × max(0.5, 1.5 - B26/5000) )
B12 (实际原料成本) = round( B10 × (1 - min(0.5, B3 × 0.00008)) × 100 ) / 100

零售 COGS   = round( (B12 + B1×0.15 + B4) × 零售销量 )
批发 COGS   = round( wsSold × B10 × 0.65 + wsSold × 0.5 )   ← 批量原料6.5折+简包装

固定成本:
  租金    = B5
  人工    = B9
  营销    = B13
  培训    = B25 × B24
  杂费    = round(0.05 × B24 × 1500)
  水电折旧 = round(B14 × 25 + B24 × 200) + 2000

B8 = B7 - 零售COGS - 批发COGS - 所有固定成本
```

### B20 — 口味/品质

```
B28 (原料品质) =  base: max(0.2, min(1.0, B10/5.0))
                 bonus: B3>3000 → min(0.25, (B3-3000)×0.00008)
                 cap: 1.0

B21 (员工满意度) — 见熵循环

B20 = round( B21 × 0.45 + B28 × 0.55, 2 )
```

## 三、状态演进（每月推演）

### FAT (疲劳度 0–100)

```
零售 UR = min(1, 零售需求 / physCap)

FAT 变化 d:
  UR > 90%:  d = round((UR - 0.90) × 40)
  UR > 95%:  d += 3                       ← 极限加班更伤
  UR < 80%:  d = -round((0.80 - UR) × 20) ← 低负荷恢复

  面积>150 且 人数 < ceil(面积/40):  d += 5  ← 人手不足惩罚

  newFAT = clamp(10, 100, FAT + d)
```

→ FAT 只跟零售 UR 挂钩。B2B 合同是计划性生产，不产生疲劳。所以低地段工厂即使走量很大，工人 FAT 也稳在 10–20。

### BRAND (知名度)

```
growth = B21 × 10 + B28 × 12 + max(0, (100-FAT)/100) × 6 + √营销 × 0.15
decay = BRAND × 0.02
growthMultiplier = max(0.05, 1 - BRAND/400)  ← 越高越慢
BRAND += growth × gm - decay - 缺货惩罚( sr × 10 )

品质惩罚 (B28 < 0.40):
  每月额外扣 gap × 25 点, 天花板锁死在 25-50
  恢复后增长打4折
```

### EMP (经验值 0–200)

```
EMP += max(1, 10 - round(EMP × 0.05))
cap: 200
```

### B21 (员工满意度 0–1)

```
payPerUnit = B9 / max(B3, 1)     ← 边际人工成本
baseline = 3 + B15 × 0.4
baseSat = payPerUnit ≥ baseline
  ? 0.7 + min((payPerUnit-baseline)/(baseline×2), 0.3)
  : payPerUnit / baseline × 0.7

overwork = max(0, B3/physCap - (0.8 + 0.2 × baseSat)) × 1.5
B21 = clamp(0, 1, baseSat - overwork)

FAT 侵蚀:  FAT>60 → B21 -= (FAT-60)×0.005
           FAT>85 → B21 -= (FAT-85)×0.012
           下限 0.15
```

## 四、三条策略路线

| | 🏠 社区 | 🏭 工厂 | ✨ 高奢 |
|---|---|---|---|
| 面积 | 30–90 m² | 150–250 m² | 70–120 m² |
| 地段 | 3–5 | 1–3 | 7–10 |
| 售价 | ¥18–26 | ¥14–20 | ¥28–38 |
| 原料 | ¥3–4 | ¥2–3 | ¥4–6 |
| 员工 | 1–4 人 | 4–10 人 | 2–4 人 |
| 收入模式 | 全零售 | 零售 + B2B + 冗余批发 | 零售 + 旅游客流 |
| 租金类型 | 商业 | 工业(折扣) | 商业(核心商圈) |

### 工厂 B2B 经济学

工厂的核心差异在 B2B 合同——**与地段无关的第二收入通道**。

每人每月接 500 件企业订单（¥18售价时批发 ¥8.10/件，原料 ¥2 时净赚 ¥5.20/件）。
3 人月出 B2B 1500 件，批净约 ¥7,800。加上工业区低租金，低地段工厂也能存活。

规模越大越依赖 B2B。零售只是"顺便卖"——工厂不靠过路客活。
